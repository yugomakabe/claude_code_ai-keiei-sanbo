**事業テーマ**: 議事録SaaS・業務効率化  
**ターゲット**: 従業員50名以下の中小企業  
**作成日**: 2026-06-27  

---

## 1. PEST分析

外部マクロ環境を4軸で整理し、事業機会と脅威を抽出する。

---

### Political（政治・法規制）

| 項目 | 内容 | 事業への影響 |
|---|---|---|
| **AI推進法（2025年施行）** | AIを「経済社会の発展の基盤」と位置づけ。罰則なしの推進型立法で、企業のAI活用を政策的に後押し | ✅ 顧客企業がAI導入に前向きになる追い風 |
| **デジタル化・AI導入補助金2026** | 旧「IT導入補助金」を名称変更。AI搭載SaaSが補助対象に明記。導入費用の1/2〜3/4を補助可能 | ✅ 中小企業の導入障壁を大幅に低下させる |
| **AI事業者ガイドライン（第1.2版）** | 経済産業省・総務省が2026年3月公表。透明性・安全性の自主ガイドラインとして整備 | ⚠️ データ取り扱いの透明性確保が必要 |
| **人工知能基本計画（2025年12月閣議決定）** | 「信頼できるAI」による日本再起を国家方針として策定。中小DX支援を重点項目に位置づけ | ✅ 公的支援の継続・拡充が期待できる |
| **データプライバシー規制の強化傾向** | EU AI Actに代表されるグローバルな規制強化。日本でも個人情報保護法の改正議論が続く | ⚠️ 会議音声データの取り扱いに注意が必要 |

**政治環境のまとめ**: 日本政府はAI活用を強力に推進しており、補助金・法整備ともに事業追い風。ただし音声・会議データのプライバシー規制は引き続き注視が必要。

### Economic（経済）

| 項目 | 内容 | 事業への影響 |
|---|---|---|
| **中小企業SaaS利用率の急上昇** | 従業員100名以下のSaaS利用率が2024年38% → 2025年48%に急増。インボイス対応・リモートワーク・生成AI普及が背景 | ✅ 市場の受容性が急速に高まっている |
| **最低賃金の継続引き上げ** | 人件費増加が経営課題となり、業務効率化・省力化への投資意欲が高まる | ✅ 「コスト削減」文脈でSaaS導入が正当化しやすい |
| **IT予算の制約** | 国内企業のIT予算は売上高比率の平均2.15%。そのうち70〜80%を既存システム維持に充てており、新規導入予算が限られる | ⚠️ 低価格・早期ROIの提示が必須 |
| **業務効率化SaaSのROI** | 導入企業の初年度ROIは100〜300%が期待値。人件費換算で年間100〜200万円の削減効果が出やすい | ✅ 経営者への定量的訴求がしやすい |
| **国内AI市場の急成長** | 2025年に2.37兆円 → 2029年に6.89兆円（CAGR 36%、IDC）。グローバルAI会議アシスタント市場は2025年 35億ドル → 2035年 342億ドル（CAGR 25.62%） | ✅ 市場全体の成長余地が大きく、早期参入の価値が高い |

**経済環境のまとめ**: 人件費上昇×SaaS受容率向上という二重の追い風。一方、中小企業の予算制約から「月1万円以上は厳しい」という価格感度に注意が必要。

### Social（社会・文化）

| 項目 | 内容 | 事業への影響 |
|---|---|---|
| **ハイブリッドワークの定着** | テレワーカー率が24.6%（令和6年度調査）。週2〜3日出社＋リモートの混在が標準化 | ✅ オンライン会議が恒常化し、録音・議事録ニーズが継続 |
| **議事録作成への高い不満** | 60%以上のビジネスパーソンがオンライン会議の議事録作成に不満（集中阻害・工数過大）。リモートワーカーの60%近くが会議情報の保持に課題を感じる | ✅ 課題の普遍性と深刻度が高く、導入動機になりやすい |
| **会議のタスク漏れ問題** | 口頭決定したタスクが担当者に伝わらず実行されない「タスク消滅問題」が中小企業で深刻。フォローアップが属人化しマネージャーの負担になっている | ✅ MeetMemoのコア訴求点（通知機能）が直撃する課題 |
| **働き方改革の継続** | 労働基準法の40年ぶり大改正が議論中。生産性向上が社会的義務として認識されつつある | ✅ 「会議の生産性」への関心が高まっている |
| **DX人材不足** | 中小企業でIT専任者がいない・兼任が多い。生成AI活用方針を策定済みが34%にとどまる | ⚠️ 導入・定着支援が手厚くないとチャーンが発生しやすい |

**社会環境のまとめ**: ハイブリッドワーク定着と議事録不満の組み合わせは強力な追い風。ただしIT担当者不在の中小企業へは「すぐ使える・すぐわかる」UXと手厚いオンボーディングが必須。

### Technological（技術）

| 項目 | 内容 | 事業への影響 |
|---|---|---|
| **音声認識精度の急向上** | 日本語文字起こし精度が98%超（Notta：98.86%）を達成。商用サービスとして十分な精度が実現済み | ✅ 商品品質の底上げが容易になりMVP開発ハードルが低下 |
| **LLMによる要約・タスク抽出** | GPT-4o・Claude・Gemini等の大規模言語モデルが会議内容のタスク抽出を高精度で実現。APIコストも急激に低下中 | ✅ コア機能（タスク抽出・通知）を低コストで構築可能 |
| **NVIDIA「Parakeet-TDT」等の新モデル台頭** | 高速・低遅延のリアルタイム音声認識モデルが登場。会議中のリアルタイム処理がより現実的に | ✅ リアルタイム通知の実現が技術的に容易になる |
| **Zoom・Teams・Meet のAPI標準化** | 主要Web会議ツールのAPIが成熟し、ボット参加・録音取得が容易に実装可能 | ✅ 既存の会議環境に追加設定なしで統合できる |
| **プラットフォームリスク（機能取り込み）** | ZoomやTeamsが議事録・要約機能を標準搭載しつつある（Copilot等）。Slack AIもサマリー機能を拡充 | ⚠️ プラットフォームへの機能取り込みで差別化が失われるリスク |
| **ボットフリーの新潮流** | Granola・Jamie等がローカル処理型を採用し、ボット参加への心理的抵抗を解消 | ⚠️ ユーザーのプライバシー意識が高まり、クラウドへの音声送信への抵抗が生じる可能性 |

**技術環境のまとめ**: LLM・音声認識の成熟でMVP開発コストは低い。一方、ZoomやTeamsによる機能取り込みは中長期の最大リスク。「タスク通知＋実行管理」という下流工程への特化で差別化を維持することが重要。

## 2. SWOT分析

PEST分析の機会・脅威を外部要因に反映し、内部の強み・弱みと統合する。

| | **強み（Strengths）** | **弱み（Weaknesses）** |
|---|---|---|
| **内部環境** | ・タスク自動抽出＋担当者通知というコア差別化機能<br>・中小企業特化の低価格設計（月3,000〜8,000円）<br>・IT担当者不在でも使えるシンプルなUX設計<br>・日本語特化チューニングによる高精度 | ・新規参入のため認知度ゼロ<br>・開発・サポートリソースが限られる<br>・セールス・CSのネットワーク未整備<br>・資金力で大手競合に劣る |

| | **機会（Opportunities）** | **脅威（Threats）** |
|---|---|---|
| **外部環境** | **P** AI推進法・デジタル化補助金でAI SaaS導入が後押し<br>**E** 中小企業SaaS利用率が38%→48%に急上昇<br>**E** 最低賃金上昇で業務効率化投資意欲が高まる<br>**S** ハイブリッドワーク定着でオンライン会議が恒常化<br>**S** 60%以上の議事録作成不満が顕在的課題<br>**T** LLM・音声認識APIのコスト低下でMVP開発が低コスト化 | **P** 音声データに関するプライバシー規制強化の可能性<br>**E** 中小企業の予算制約・「SaaSポカリプス」（統廃合）<br>**T** Zoom/Teams/Copilotが議事録機能を標準搭載<br>**T** 競合（Fireflies等）がタスク通知機能を追加<br>**S** AI GIJIROKUのようなサービス終了事例によるユーザーの信頼不安 |

### クロスSWOT戦略

| 戦略 | 内容 |
|---|---|
| **SO戦略**（強みで機会を活かす） | 補助金対象製品として申請サポートを行い、価格障壁を下げながら中小企業へのリーチを加速する |
| **WO戦略**（弱みを補いつつ機会を活かす） | 代理店・会計士・社労士ネットワークと提携し、IT専任不在の中小企業へ間接的にリーチする |
| **ST戦略**（強みで脅威に対抗する） | プラットフォームに取り込まれにくい「タスク実行管理」機能（進捗追跡・次回会議サマリー）に注力し、議事録ツールの上位レイヤーへとポジション移動 |
| **WT戦略**（弱みと脅威の最小化） | 月額契約のみ・初期費用ゼロ設計でサービス終了リスクへの不安を軽減。データエクスポート機能を標準搭載 |

## 3. 3C分析

### Customer（顧客）

#### セグメント定義
- **対象**: 従業員50名以下の中小企業
- **主要業種**: IT・コンサル・士業・不動産・建設・製造の管理部門
- **意思決定者**: 社長・役員、または総務・経営企画担当者

#### 顧客の課題構造

```
表層的な課題：「議事録を作るのが面倒・時間がかかる」
        ↓
本質的な課題：「会議で決まったことが実行されない」
              「誰が何をやるかが曖昧なまま次の会議になる」
              「マネージャーがフォローアップに追われる」
```

#### 顧客インサイト

| ニーズ | 具体的内容 |
|---|---|
| **即導入性** | IT担当者不在のため、設定・運用が複雑なツールは敬遠される。「Slack連携して3分で始まった」体験が必要 |
| **日本語サポート** | 英語UIや英語サポートは導入障壁。日本語UIと日本語チャットサポートが必須 |
| **価格の納得感** | 月1万円以上は稟議が通りにくい。月3,000〜5,000円帯かつ「何時間分の人件費削減に相当するか」の提示が有効 |
| **補助金活用** | デジタル化・AI導入補助金の対象であることが導入の後押しになる |
| **タスク漏れの解消** | 「あの件どうなった？」の確認工数をゼロにしたい。担当者が自動で通知を受け取ることで責任の可視化が実現する |

#### 顧客の購買プロセス（中小企業）

```
問題認識（議事録の属人化・タスク漏れ）
→ 情報収集（Google検索・知人紹介・SaaS比較サイト）
→ 無料トライアル（1〜2週間で効果を体感）
→ 社内説得（社長に見せる・補助金申請）
→ 本導入・定着（オンボーディング支援が離脱を防ぐ）
```

| 日本語サポート | ○ | ✗ | ◎ | ◎ |
| 補助金対応 | △ | ✗ | △ | ○（申請サポート含む） |

#### 競合の共通的な弱点（参入機会）

1. **タスクの「通知」が自動化されていない**: 抽出はできてもSlackやメールへのPush通知は手動設定・手動実行が多い
2. **実行管理がない**: 議事録を作った後の「誰がどこまでやったか」の追跡機能が全競合に欠ける
3. **中小企業向けチャネルが弱い**: 大手SaaS経由の比較・自社検索が中心で、代理店・士業ネットワーク経由の販路がほぼ空白
4. **オンボーディング支援が薄い**: IT担当不在の中小企業が使いこなすための伴走支援が不足

### Company（自社）

#### 提供価値の整理

> **MeetMemoのコアバリュー**  
> 「会議が終わった瞬間に、全員のToDo通知が完了する」

| レイヤー | 機能 | 競合との差異 |
|---|---|---|
| **Layer 1（基盤）** | 音声録音・文字起こし（日本語精度98%超） | 競合と同水準 |
| **Layer 2（差別化）** | AIによるタスク自動抽出（誰が・何を・いつまでに） | 競合より優位 |
| **Layer 3（独自価値）** | 担当者への自動通知（Slack/Teams/メール） | 競合にない機能 |
| **Layer 4（将来）** | タスク進捗追跡・次回会議の自動サマリー | 競合にない機能 |

#### 自社リソースと強み

| リソース | 現状 | 強化方針 |
|---|---|---|
| **技術** | Claude API・Whisper APIを活用したLLM開発が可能 | Layer 3・4の早期実装を優先 |
| **市場知識** | 先行事業検証レポートで課題・競合を調査済み | ターゲット企業20社へのインタビューを実施 |
| **価格競争力** | 中小企業向け月3,000〜8,000円の価格設計 | IT導入補助金の対象登録を申請 |
| **差別化機能** | タスク通知という競合空白地帯を確保 | Slack Botによるβ版プロトタイプ優先 |

#### 課題と対応方針

| 課題 | 対応方針 |
|---|---|
| 認知度がゼロ | コンテンツマーケティング（「タスク漏れ防止」ブログ）＋比較サイト掲載 |
| サポートリソース不足 | オンボーディング動画の整備＋チャットサポート（Intercom等） |
| 販路がない | 税理士・社労士・中小企業診断士との紹介パートナーシップ |
| 機能追いつかれリスク | タスク実行管理・進捗追跡への先行投資でモート（堀）を構築 |

## 総括：戦略提言

### 1. ポジショニングの確定

> **「議事録ツール」ではなく「会議を実行力に変えるエンジン」として定義する**

既存競合はすべて「議事録の作成・共有」でチェーンが終わっている。MeetMemoは「タスクの通知・追跡」まで一気通貫することでカテゴリを再定義する。

### 2. 初期ターゲットの絞り込み

```
ベストフィット顧客像（ICP：Ideal Customer Profile）
─────────────────────────────────
・従業員10〜30名の成長フェーズにある中小企業
・週3〜5回のオンライン会議がある（Zoom/Teams利用中）
・マネージャーが「タスク漏れ・追いかけ疲れ」を感じている
・IT専任者はいないが、社長または管理部門が決済権限を持つ
```

### 3. 3ヶ月のアクションプラン

| フェーズ | 期間 | 目標 |
|---|---|---|
| **検証** | 〜1ヶ月 | ICP企業20社インタビュー。「タスク通知」への課金意欲を確認 |
| **MVP** | 〜3ヶ月 | Slack Bot型プロトタイプ構築。β企業10社で継続率・NPS測定 |
| **PMF確認** | 〜6ヶ月 | 月3,000円プランで30社課金。チャーン率10%以下を目標 |
